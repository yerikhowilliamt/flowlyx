## 4. Pull Request Description
This PR implements the global search functionality required by Phase 28. Following the ponytail lazy mode, it avoids over-engineering by using a simple concurrent queries approach (`Promise.all` with ILIKE queries over Tasks, Projects, Workspaces) instead of requiring a separate Search Engine infrastructure. It implements a unified `/search` endpoint returning grouped results.

## 5. Summary
- Created `SearchQueryDto` for query validation and configuration (limit, search string).
- Created `SearchService` to handle parallel queries to Tasks, Projects, and Workspaces.
- Created `SearchController` to expose the REST API endpoint.
- Added `SearchModule` to `AppModule`.
- Updated error documentation for Phase 28 to log the trade-offs of this approach.
