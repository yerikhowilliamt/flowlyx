## Description
This PR implements the Email Service module for Flowlyx as part of Phase 37. It integrates `nodemailer` with `BullMQ` to enable asynchronous email delivery, avoiding blockages on the main API thread.

## Summary
- Added `nodemailer` and `@types/nodemailer` as dependencies.
- Added validation for SMTP configuration (`SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, `SMTP_FROM`) inside `env.validation.ts`.
- Created `EmailModule`, `EmailService`, and `EmailProcessor`.
- Configured a background worker (`email-queue`) to process outgoing emails.
- Added full test coverage for the Email module.
- Resolved and documented linter issues related to strict `any` typing in `docs/errors/phase 37 - email service.md`.
