# Phase 38: Scheduled Jobs

## Summary
Implemented the Scheduled Jobs module as required by Phase 38, fully compliant with the Flowlyx Engineering Handbook. This module utilizes BullMQ to handle repeatable (cron) jobs efficiently across distributed environments, leveraging the existing Redis infrastructure.

## Implementation Details
- **Module Foundation**: Created `ScheduledJobsModule` to host and expose scheduled background tasks.
- **Service Integration**: Implemented `ScheduledJobsService` which allows adding, fetching, and removing repeatable jobs programmatically.
- **Robust Processing**: Developed `ScheduledJobsProcessor` mapped to `SCHEDULED_JOBS_QUEUE` to catch and process the scheduled tasks securely.
- **Heartbeat Example**: Instantiated a default `system-heartbeat` job that runs every 5 minutes to prove the infrastructure works and logs correctly.
- **Logging**: Configured all operations with `PinoLogger` for structured tracking.
